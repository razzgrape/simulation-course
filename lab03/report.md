## Описание модели

### Сетка и состояния клеток

Модель представляет собой двумерный клеточный автомат на прямоугольной сетке размером `rows × cols` (по умолчанию 100×140). Используется **окрестность Мура** — каждая клетка взаимодействует с 8 соседями (по горизонтали, вертикали и диагонали).

Каждая клетка находится в одном из пяти состояний:

```python
class CellState(IntEnum):
    EMPTY = 0    # пустая земля
    TREE = 1     # дерево
    BURNING = 2  # горит
    WATER = 3    # вода (преграда)
    BURNT = 4    # выгоревшая (промежуточное состояние)
```

Состояние `BURNT` — промежуточное между горением и пустотой. Без него огонь исчезал бы мгновенно, что нереалистично. С ним клетка визуально «догорает» несколько шагов.

### Соседи

Восемь соседей определены как смещения `(dr, dc)` с индексом направления для расчёта ветра:

```python
NEIGHBORS = [
    (-1,  0, 0),  # N
    (-1,  1, 1),  # NE
    ( 0,  1, 2),  # E
    ( 1,  1, 3),  # SE
    ( 1,  0, 4),  # S
    ( 1, -1, 5),  # SW
    ( 0, -1, 6),  # W
    (-1, -1, 7),  # NW
]
```

Третье число — индекс направления от 0 (север) до 7 (северо-запад) по часовой стрелке. Он нужен для того, чтобы при обработке ветра определить, совпадает ли направление распространения огня с направлением ветра.

### Конфигурация

Все параметры собраны в dataclass-ах, которые GUI может изменять в реальном времени:

```python
@dataclass
class SimulationConfig:
    rows: int = 80
    cols: int = 120
    tree_grow_prob: float = 0.01       # p — вероятность роста дерева
    lightning_prob: float = 0.0001     # f — вероятность спонтанного возгорания
    base_spread_prob: float = 0.85     # базовая вероятность распространения огня
    humidity: float = 0.0              # 0..1, снижает вероятность распространения
    wind: WindConfig = ...
    slope_factor: float = 0.2          # множитель влияния уклона
    burn_duration: int = 2             # сколько шагов клетка горит
```

### Шаг симуляции

На каждом шаге новое состояние **всех** клеток вычисляется одновременно. Для этого создаётся копия сетки, к копии применяются правила, и затем она заменяет оригинал:

```python
def step(self):
    new_grid = self.grid.copy()
    new_burn = self.burn_timer.copy()
    rng = np.random.random((self.rows, self.cols))
    # ... все правила применяются к new_grid ...
    self.grid = new_grid
    self.burn_timer = new_burn
```

Матрица `rng` генерируется одна на весь шаг — каждой клетке соответствует число от 0 до 1, которое сравнивается с порогами вероятностей. Это проще и быстрее, чем вызывать генератор для каждой проверки отдельно.

## Основные правила

Четыре базовых правила взяты из слайда 12 презентации.

### Правило 1. Горящая клетка выгорает

У каждой горящей клетки есть таймер (`burn_duration`). На каждом шаге он уменьшается на 1, и когда доходит до нуля, клетка переходит в `BURNT`. На следующем шаге `BURNT` переходит в `EMPTY`:

```python
if state == CellState.BURNING:
    new_burn[r, c] -= 1
    if new_burn[r, c] <= 0:
        new_grid[r, c] = CellState.BURNT

elif state == CellState.BURNT:
    new_grid[r, c] = CellState.EMPTY
```

### Правило 2. Распространение огня

Для каждой клетки с деревом обходим все 8 соседей. Если хотя бы один горит — вычисляем вероятность поджога через `_get_spread_probability` и сравниваем с `rng`. При успехе дерево загорается, дальше соседей не проверяем:

```python
elif state == CellState.TREE:
    ignited = False
    for dr, dc, dir_idx in NEIGHBORS:
        nr, nc = r + dr, c + dc
        if 0 <= nr < self.rows and 0 <= nc < self.cols:
            if self.grid[nr, nc] == CellState.BURNING:
                spread_dir = OPPOSITE_DIR[dir_idx]
                prob = self._get_spread_probability(nr, nc, r, c, spread_dir)
                if rng[r, c] < prob:
                    new_grid[r, c] = CellState.BURNING
                    new_burn[r, c] = self.config.burn_duration
                    ignited = True
                    break
```

Здесь `spread_dir = OPPOSITE_DIR[dir_idx]` — важный момент. `dir_idx` — это направление **от текущего дерева к горящему соседу**. Но для расчёта ветра нужно направление **распространения огня**, то есть от горящего к дереву — обратное направление. Словарь `OPPOSITE_DIR` даёт это:

```python
OPPOSITE_DIR = {0: 4, 1: 5, 2: 6, 3: 7, 4: 0, 5: 1, 6: 2, 7: 3}
```

### Правило 3. Спонтанное возгорание (молния)

Если дерево не поджёг ни один сосед, оно может загореться случайно с малой вероятностью `f`:

```python
    if not ignited:
        if rng[r, c] < self.config.lightning_prob:
            new_grid[r, c] = CellState.BURNING
            new_burn[r, c] = self.config.burn_duration
```

Без этого правила после того как все пожары потухнут, лес бесконечно растёт без динамики. Молния запускает новые пожары и поддерживает цикличность системы.

### Правило 4. Рост деревьев

Пустая клетка с вероятностью `p` превращается в дерево:

```python
elif state == CellState.EMPTY:
    if rng[r, c] < self.config.tree_grow_prob:
        new_grid[r, c] = CellState.TREE
```

Это замыкает жизненный цикл: лес растёт → загорается → выгорает → снова растёт.

## Дополнительные правила

Четыре дополнительных правила реализованы(погодные и топографические факторы). Все они собраны в метод `_get_spread_probability`, который модифицирует базовую вероятность распространения огня для конкретной пары «горящая клетка → соседнее дерево»:

```python
def _get_spread_probability(self, from_r, from_c, to_r, to_c, dir_idx) -> float:
    prob = self.config.base_spread_prob
    # ... модификации prob ...
    return np.clip(prob, 0.0, 1.0)
```

`np.clip` в конце обрезает результат в `[0, 1]`, потому что сумма бонусов от ветра и уклона может вывести вероятность за допустимый диапазон.

### Доп. правило 1. Влажность

Высокая влажность снижает вероятность распространения:

```python
prob *= (1.0 - self.config.humidity * 0.8)
```

При `humidity = 0` — множитель равен 1, ничего не меняется. При `humidity = 1` — множитель 0.2, вероятность падает на 80%. Полностью огонь не блокируется, но распространяется значительно медленнее.

### Доп. правило 2. Ветер

Сравниваем направление распространения огня (`dir_idx`) с направлением ветра. По ветру — бонус, под углом — половина бонуса, против ветра — штраф:

```python
if self.config.wind.enabled and self.config.wind.strength > 0:
    wind_dir = self.config.wind.direction
    if dir_idx == wind_dir:
        prob += self.config.wind.strength                    # точно по ветру
    elif dir_idx == (wind_dir + 1) % 8 or dir_idx == (wind_dir - 1) % 8:
        prob += self.config.wind.strength * 0.5              # под углом 45°
    elif dir_idx == OPPOSITE_DIR[wind_dir]:
        prob -= self.config.wind.strength * 0.6              # прямо против
    elif dir_idx == (OPPOSITE_DIR[wind_dir] + 1) % 8 or \
         dir_idx == (OPPOSITE_DIR[wind_dir] - 1) % 8:
        prob -= self.config.wind.strength * 0.3              # под углом 45° против
```

Операция `% 8` нужна для закольцовки направлений: направление 7 (СЗ) + 1 = 0 (С). Это создаёт характерную вытянутую форму пожара по направлению ветра.

### Доп. правило 3. Уклон местности

Берётся разность высот между целевой и исходной клеткой:

```python
if self.elevation is not None:
    dh = self.elevation[to_r, to_c] - self.elevation[from_r, from_c]
    prob += dh * self.config.slope_factor
```

Если `dh > 0` (огонь идёт вверх по склону), вероятность увеличивается. Вниз — уменьшается. Физическое обоснование: горячий воздух поднимается и предварительно нагревает деревья выше по склону.

Карта высот генерируется при создании симуляции — случайный шум на грубой сетке (каждая 8-я клетка), затем билинейная интерполяция через `scipy.ndimage.zoom` для плавности, и нормализация в `[0, 1]`:

```python
def _generate_elevation(self) -> np.ndarray:
    noise = np.random.randn(self.rows // 8 + 1, self.cols // 8 + 1)
    from scipy.ndimage import zoom
    elevation = zoom(noise, (self.rows / (self.rows // 8 + 1),
                              self.cols / (self.cols // 8 + 1)))
    elevation = elevation[:self.rows, :self.cols]
    elevation = (elevation - elevation.min()) / (elevation.max() - elevation.min() + 1e-9)
    return elevation
```

### Доп. правило 4. Преграды (вода)

Клетки с состоянием `WATER` не обрабатываются ни одной веткой `if/elif` в методе `step()`. Они не горят, не зарастают деревьями, не выгорают — просто сохраняют своё состояние навсегда. Огонь не может перейти на воду, потому что проверка `self.grid[nr, nc] == CellState.BURNING` не сработает для водной клетки.

Реки генерируются с синусоидальным смещением для извилистости:

```python
def add_water_river(self, col: int, width: int = 2):
    for r in range(self.rows):
        for dc in range(width):
            c = col + dc + int(2 * np.sin(r / 5.0))
            if 0 <= c < self.cols:
                self.grid[r, c] = CellState.WATER
```

Это разбивает лес на изолированные зоны, где пожар в одной зоне не может перекинуться на другую.

## Вывод

В ходе лабораторной работы был реализован двумерный клеточный автомат для моделирования лесных пожаров на Python с графическим интерфейсом.
Модель включает 4 основных правила (выгорание, распространение огня на соседей, спонтанное возгорание от молнии, рост новых деревьев) и 4 дополнительных (влажность, ветер, уклон местности, водные преграды). Все дополнительные правила влияют на единую величину — вероятность распространения огня от горящей клетки к соседнему дереву — через метод _get_spread_probability, что позволяет им комбинироваться и взаимодействовать друг с другом.
